## 4. DRY - Don‚Äôt Repeat Yourself
Avoid duplication. Build reusable systems. Every effort should contribute to long-term infrastructure through modular code, templates, structured data, and automated processes.

**Equation**

**Implications**  
_Let your work compound over time._  
- Evaluate how every project can contribute to some form of resource that is useful across other projects (now and future); 
- Use automation to reduce manual repetition of boring predictable protocols;
- Design templates for everything: projects, notes, documents, presentations, code, etc.; 
- Write reusable code by writing generic modules, packages and libraries;  
- Design a system for store, retrieve and update datasets so you don't need to download and organize it every time;  

**Moderation**

- Avoid overbuilding before patterns emerge;

- Improve systems iteratively, not all at once;

**Equation**

**Implications**

- Favor modular structures over tangled systems;  
- Write code and docs that are clear without deep explanation;  
- Choose tools that are open, inspectable, and adaptable;  
- Minimize interface complexity while maximizing clarity;

**Moderation**  
_Simplicity is a guide ‚Äî not a limit._

- Use sophistication with intention, not ego;

- Use notes and comments strategically;  
- Apply standard naming rules and automate documentation whenever possible;  
- Preserve context, logic, and instructions for fast reentry;  
- Maintain documentation with updates and clarity;

**Moderation**  
_Documentation must evolve ‚Äî not suffocate._

- Focus documentation on reuse, not bureaucracy;  
- Avoid documenting the obvious;  
- Clarify what matters ‚Äî revise as systems mature;

**Implications**
